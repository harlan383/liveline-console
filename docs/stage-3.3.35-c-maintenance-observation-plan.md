## 回退链路保留策略

- 短期内继续保留 `gost` 8443。
- 不建议立即关闭 `gost` 8443。
- 不建议立即替换 `gost` 8443。
- 不建议让 `socat` 接管 8443。
- 如未来要清理回退链路，必须单独进入审批阶段。
- 回退链路清理前必须确认 `socat` 18443 长期稳定性、客户端回退方式和业务低风险窗口。

## 未来变更审批边界

- 如未来需要清理 `gost` 8443，必须单独进入审批阶段。
