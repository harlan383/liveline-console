## 正式执行流程

### Phase 0：执行前人工确认

- 确认 Stage 3.3.29 的 C-minimal Go 仍有效。
- 确认本 runbook final 已合并并被人工确认。
- 确认本次执行不扩大到 8443 接管、`gost` 8443 变更、远程命令或新增端口。
- 确认本次执行只允许修改 `node.share_link` 到已验收的 `socat` 18443 候选链路。

### Phase 1：备份旧 `node.share_link`

- 修改前必须备份旧值。
- 备份过程不得把完整链接写入文档、日志、任务结果或报告。
- 备份位置、访问权限和恢复步骤必须由执行人和回滚负责人确认。

### Phase 2：确认新候选链路来自 `socat` 18443

- 新候选链路必须来自已验收的 `socat` 18443 候选链路。
- 不允许使用未验收端口。
- 不允许新增监听端口。
- 不允许让 `socat` 接管 8443。
- 不允许关闭、停用、降级或替换 `gost` 8443。

### Phase 3：按批准范围修改 `node.share_link`

- 仅允许在正式执行阶段按批准范围修改 `node.share_link`。
- 修改目标只能是已验收的 `socat` 18443 候选链路。
- 修改记录不得包含完整链接。
- 修改异常时必须立即恢复旧值。

### Phase 4：客户端验收

- 完成 `node.share_link` 修改后立即执行客户端验收。
- Shadowrocket、v2rayN、软路由三类关键场景必须验收。
- 验收记录不得包含完整节点链接。
- 任一关键场景失败时进入回滚。

### Phase 5：观察期

- 观察普通网页访问、目标平台访问和直播影响。
- 出现明显异常时进入回滚。
- 观察期内 `gost` 8443 必须保持可回退。

### Phase 6：完成记录或回滚

- 验收全部通过时，记录最终结论。
- 任一停止条件触发时，立即执行回滚。
- 完成记录或回滚记录均不得包含完整节点链接、SSH Key、Passphrase、token、服务器密码或 Reality privateKey。

## 回滚方案

- `node.share_link` 修改异常时恢复旧值。
- Shadowrocket / v2rayN / 软路由任一关键场景失败时恢复旧值。
- 普通网页或目标平台访问异常时恢复旧值。
- 直播出现明显异常时恢复旧值。
- `socat` 18443 不可用时恢复 `gost` 8443 或原正式链路。
- 回滚后必须重新验证客户端可用性。
- 回滚记录不得包含完整节点链接、SSH Key、Passphrase、token、服务器密码或 Reality privateKey。

## 停止条件

- 旧 `node.share_link` 未备份。
- 新候选链路未确认。
- 执行窗口未确认。
- 执行人、验收人、回滚负责人未确认。
- 客户端无法快速回退。
- 发现完整节点链接、SSH Key、Passphrase、token、服务器密码或 Reality privateKey 写入风险。
- 发现需要接管 8443、关闭 `gost`、修改防火墙、重启服务等超出 C-minimal 范围的操作。

- 香港中转机 TCP 8443 / 18443 云安全组已放行。
- 独立云防火墙为 `Not applicable`。
- 服务器防火墙未发现本地阻塞。
- 本执行方案不新增端口。
- 本执行方案不让 `socat` 接管 8443。
- 本执行方案不关闭、停用、降级或替换 `gost` 8443。
- 凡未来涉及新增、变更监听端口或 8443 接管，仍必须重新检查云服务器安全组 / 云防火墙 / 服务器防火墙是否放行对应 TCP 端口。
