## 回滚方案确认

正式 cutover 前必须确认以下回滚方案：

- 如果 `socat` 18443 候选链路失败，继续使用 `gost` 8443 或原正式链路。
- 如果 `node.share_link` 修改后异常，应恢复旧值。
- 如果 `socat` 接管 `8443` 后异常，应恢复 `gost` 8443。
- 如果远端服务异常，应停止切换并恢复原状态。
- 如果云安全组或服务器防火墙导致端口不可达，应停止 cutover 并恢复回退链路。
- 回滚后必须再次验证客户端可用性。
- 回滚记录不得包含完整节点链接、SSH Key、Passphrase、服务器密码、token 或 Reality privateKey。
