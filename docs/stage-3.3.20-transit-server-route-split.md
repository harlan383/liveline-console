# Stage 3.3.20 Transit Server / Route Split

## 本阶段目标

本阶段将原先合并在“中转服务器”页面里的资源管理与转发关系管理拆分为两个前端页面：

- “中转服务器”：只管理中转 VPS 资源记录。
- “中转链路”：只管理“中转服务器监听端口 -> 落地服务器节点目标端口”的转发关系。

本阶段是前端页面职责拆分，不是 Worker 阶段，不是远程执行阶段，也不是正式 cutover。

## 左侧导航结构

本阶段后的左侧菜单顺序为：

1. 总览
2. 中转服务器
3. 落地服务器
4. 中转链路
5. 任务中心
6. 诊断工具
7. 设置

“节点”一级菜单继续保持移除；节点仍属于落地服务器下级记录。

## 页面职责

### 中转服务器

“中转服务器”页面只展示和维护中转 VPS 资源记录，主表列为：

- 名称
- IP 地址
- 端口
- 状态
- 操作

该页面可新增或编辑本地中转服务器资源记录。重新检测、删除、Worker 接入、远程清理仍提示为后续 Worker / API 阶段开放。

### 中转链路

“中转链路”页面展示已有转发关系，并提供本地规划入口。表格列为：

- 链路名称
- 中转服务器
- 监听端口
- 落地服务器
- 目标节点
- 目标端口
- 转发方式
- 状态
- 角色标识
- 操作

该页面保留旧高级链路规划、只读预检、历史诊断工具，但默认折叠在高级工具区。

由于中转链路表格字段较多，页面为该表格增加了专用横向滚动容器和最小宽度，避免“角色标识 / 操作”列被挤压或裁切。数据行严格对齐表头顺序，“目标端口”和“转发方式”分列展示，避免 `socat` / `gost` 错位到目标端口列。操作列固定在右侧，横向滚动时仍保持可见。链路名称、中转服务器、落地服务器、目标节点和角色标识等长文本使用省略显示，并通过 `title` 保留完整内容查看能力。

## 添加中转链路弹窗

添加中转链路弹窗只做本地规划，不创建真实线路。字段包括：

- 链路名称
- 中转服务器
- 转发方式
- 中转监听端口
- 目标落地服务器
- 目标节点
- 目标端口
- 备注

选择目标节点后，目标端口会根据节点端口自动带出；用户仍可在本地规划中调整目标端口。

## 转发方式

本阶段继续保留：

- socat
- gost

本阶段不新增 HAProxy。HAProxy TCP 如需加入，应在后续单独阶段评估。

## Worker 边界

本阶段不实现 Worker，不安装 Worker，不生成 Worker token，不提供 curl | bash 接入。

真正远程检测、远程创建、远程删除、远程清理、Worker 安装和 token 生成，都需要后续 Worker / API 阶段单独审批。

## API 使用与不足

本阶段复用现有本地 API：

- `GET /api/transit-resources?resource_type=server`
- `POST /api/transit-resources`
- `PATCH /api/transit-resources/{id}`
- `GET /api/transit-routes`
- `GET /api/nodes`

已发现但不阻塞本阶段的后续 API 不足：

- 安全删除中转服务器，需要先完成远程 socat / gost 清理。
- 删除转发链路，需要远程服务清理成功后再删除系统记录。
- 无副作用远程重检，需要 Worker 只读能力。
- Worker 接入 token / 安装流程尚未实现。

本阶段没有新增后端 API，也没有硬编码假中转服务器或假链路数据。

## 安全边界

本阶段未修改后端核心逻辑。

本阶段未修改 `node.share_link`。

本阶段未新增数据库迁移。

本阶段未新增监听端口。

本阶段未执行 SSH / 远程命令。

本阶段未连接远程服务器。

本阶段未触发后端任务。

本阶段未创建真实中转链路。

本阶段未创建真实节点。

本阶段未执行正式 cutover。

本阶段未关闭、停用、降级或替换 `gost 8443`。

本阶段未让 `socat` 接管 `8443`。

本阶段未覆盖当前 `socat 18443` 正式链路。

## 当前链路基线

- 当前正式链路：`socat 18443`
- 当前回退链路：`gost 8443`
- `node.share_link` 已指向 `socat 18443`
- 新增或变更监听端口前，必须同步检查云服务器安全组 / 云防火墙 / 服务器防火墙是否放行对应 TCP 端口

- 长字段使用省略显示，不撑爆表格。
- 添加中转链路弹窗只做本地规划。
- 目标端口可根据目标节点自动带出。
- 旧高级中转工具默认折叠并迁移到“中转链路”页面。
- 页面不默认暴露完整节点链接。
- `socat` / `gost` 保留。
- 未新增 HAProxy。
- 未实现 Worker。
- 未执行 SSH / 远程命令。
- 未修改 `node.share_link`。
- 未新增数据库迁移。
- 未新增监听端口。
