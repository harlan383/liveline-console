# Stage 3.3.19 Transit Server UI Table Alignment

## 本阶段目标

本阶段在做 Worker 第一版之前，先把前端“中转服务器”页面对齐为类似“落地服务器”的管理表格体验。

本阶段不是 Worker 实现阶段，不是真实远程执行阶段，不是正式 cutover。

## 为什么先做 UI 对齐

当前左侧菜单已经调整为“总览 / 中转服务器 / 落地服务器 / 任务中心 / 诊断工具 / 设置”。落地服务器页面已经是管理表格结构，而中转服务器页面仍偏向旧的中转链路调试界面。

为了避免用户误以为进入页面就会创建线路、执行 SSH 或修改正式链路，本阶段把中转服务器页面默认入口改为管理表格，并把高级链路规划、只读预检和历史诊断工具放入折叠区。

## 中转服务器页面新结构

页面顶部：

- 标题：中转服务器
- 说明：管理中转服务器及其转发链路；资源记录不等于真实线路。
- 按钮：添加中转服务器

页面主体：

- 中转服务器管理表格
- 默认折叠的“查看中转服务器安全说明”
- 默认折叠的“查看高级链路规划 / 只读预检 / 历史诊断工具”

## 中转服务器表格列

主表列为：

- 名称
- IP 地址
- 端口
- 状态
- 操作

主行端口显示 SSH 端口。当前后端没有独立的中转服务器 SSH 在线状态字段，因此状态显示为“未检测”或“已停用”，并标明状态来源是本地资源状态 / 未做 SSH 检测。

## 下级转发链路行设计

每台中转服务器下方展示现有转发链路下级行，视觉上使用缩进和较浅背景与主行区分。

下级行展示：

- 链路名称
- 转发方式：socat / gost
- 监听端口
- 目标地址
- 目标端口
- 状态
- 角色标识：正式链路 / 回退链路 / 只读线路

下级行操作：

- 查看：打开只读详情弹窗
- 诊断：当前只提示后续 Worker / 授权阶段开放
- 删除：当前只提示需先完成远程清理，后续阶段开放

## 和落地服务器页面的一致性

本阶段复用落地服务器页面的表格、主行 / 下级行、状态标签、弹窗、空状态和折叠安全提示视觉风格。

中转服务器页面默认不再一进入就展示大面积链路配置表单。旧的显式创建 / 诊断 / 只读预检工具仍保留，但被折叠到高级区。

## 添加中转服务器

“添加中转服务器”弹窗使用现有 `POST /api/transit-resources` 本地资源 API。

弹窗只保存本地资源记录，不安装 Worker，不生成 Worker token，不执行 SSH，不连接远程服务器。

字段包括：

- 中转服务器名称
- 公网 IP
- SSH 端口
- SSH 用户名
- 服务商 / 备注来源
- 备注

备注中不得写入 SSH Key、密码、token 或完整节点链接。

## 添加转发链路

“添加转发链路”弹窗当前只做本地规划展示。

本阶段不调用真实创建接口，不创建真实中转链路，不新增监听端口，不执行远程命令。

弹窗字段包括：

- 链路名称
- 转发方式：socat / gost
- 中转监听端口
- 目标落地服务器 / 节点
- 目标端口
- 备注

端口提醒保留：新增或变更监听端口后，必须同步检查云服务器安全组 / 云防火墙 / 服务器防火墙是否放行对应 TCP 端口。

## 删除 / 编辑 / 重新检测处理

- 编辑中转服务器：使用现有 `PATCH /api/transit-resources/{id}` 更新本地资源记录。
- 重新检测：当前没有无副作用 Worker 接口接入默认按钮，只提示后续 Worker / API 阶段开放。
- 删除中转服务器：当前不删除系统记录。未来必须先远程清理 socat / gost 成功后，才允许删除系统记录；如果远程清理失败，应禁止删除并展示失败原因。
- 删除转发链路：当前没有安全删除 API 接入默认按钮，只提示后续阶段开放。

## socat / gost 保留

本阶段继续保留：

- socat
- gost

socat 适合临时测试和简单转发；gost 作为兼容 / 备用转发方式。

本阶段不新增 HAProxy。HAProxy TCP 后续单独阶段再评估加入。

## API 现状与不足

当前已可使用：

- `GET /api/transit-resources`
- `POST /api/transit-resources`
- `PATCH /api/transit-resources/{id}`
- `GET /api/transit-routes`

当前不在本阶段接入默认按钮：

- 远程重新检测接口，因为它需要 SSH 临时凭据并会创建后端任务。
- 真实中转链路创建接口，因为它会创建远程转发任务。

当前缺少或未接入：

- 无副作用的中转服务器在线状态字段
- 安全删除中转服务器 API
- 安全删除转发链路 API
- Worker 接入 token / 安装流程
- 远程清理成功后再删除系统记录的任务流程

因此，本阶段没有硬编码假状态，没有伪造远程能力，也没有新增后端接口。

- 未执行 SSH / 远程命令。
- 未连接远程服务器。
- 未创建真实中转链路。
- 未实现 Worker。
- 未安装 Worker。
- 未生成 Worker token。
