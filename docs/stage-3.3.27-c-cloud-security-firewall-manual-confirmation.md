本阶段记录 C 方案正式 cutover 前的云服务器安全组 / 云防火墙 / 服务器防火墙人工确认项。本阶段不是正式 cutover，不执行 SSH，不执行远程命令，不读取或修改 `node.share_link`，不触发后端任务，不新增数据库迁移，不新增监听端口，不修改防火墙规则，不让 `socat` 接管 8443，不关闭、停用、降级或替换 `gost` 8443。

## 当前阶段结论

- 本阶段是云服务器安全组 / 云防火墙 / 服务器防火墙人工确认记录阶段。
- 本阶段不是正式 cutover。

- 服务器侧只读预检结论为 `Ready`。
- 云服务器安全组 / 云防火墙人工确认项已完成。

- 当前服务商后台未发现独立云防火墙功能，按 `Not applicable` 记录。
- 整体 C 方案仍为 `Blocked`。
- 正式 cutover 仍为 `No-Go`。

## 端口与 cutover 关系说明

- 8443 当前由 `gost` 监听，仍是正式 / 回退链路。
- 18443 当前由 `socat` 监听，仍是候选链路。
- 本阶段不允许 `socat` 接管 8443。
- 本阶段不允许关闭、停用、降级或替换 `gost` 8443。
- 本阶段不新增监听端口。
- 本阶段不修改任何云侧或服务器侧防火墙规则。
- 本阶段不修改 `node.share_link`。

## 强制提醒

- 凡涉及新增、变更监听端口或 8443 接管，必须同步检查云服务器安全组 / 云防火墙 / 服务器防火墙是否放行对应 TCP 端口。
- 即使服务器侧监听正常，云安全组或云防火墙未放行也会导致外部不可达。
- 如果未来进入正式 cutover，仍必须重新确认执行窗口、回滚方案、负责人、验收人、`node.share_link` 修改权限、远程命令权限、8443 接管权限和 `gost` 8443 变更权限。
