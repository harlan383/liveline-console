# Stage 3.3.23 C Read-Only Preflight Execution Approval

本阶段只新增 C 方案真实只读预检执行申请 / 审批文档。本阶段不是正式 cutover，不 SSH，不连接服务器，不执行远程命令，不触发后端任务，不读取或修改正式节点链接。

## 当前阶段结论

- 本阶段是 C read-only preflight execution approval。
- 本阶段不是正式 cutover。
- 当前仍保持 No-Go。
- 本阶段只写真实只读预检执行申请 / 审批文档。
- 本阶段不执行任何命令。
- 本阶段不 SSH。
- 本阶段不远程。
- 本阶段不读取或修改 `node.share_link`。
- 本阶段不切换端口。
- 本阶段不做 cutover。
- 本阶段不新增数据库迁移。
- 本阶段不新增监听端口。
- 本阶段不关闭、停用或降级 `gost` 8443。
- 本阶段不让 `socat` 接管 `8443`。

## 与 Stage 3.3.22 的关系

- Stage 3.3.22 已合并所有“只审批不执行”的预检边界。
- Stage 3.3.23 是在 Stage 3.3.22 的基础上提出真实只读预检执行申请。
- 当前申请结论仍为 No-Go。
- 当前不授权执行任何远程只读命令。
- 当前不授权 SSH 登录。
- 当前不授权读取或修改 `node.share_link`。
- 当前不授权正式 cutover。

## 未来如需把真实只读预检改为 Go

未来如需把真实只读预检改为 Go，必须补齐：

- 明确 SSH 登录是否批准。
- 明确远程只读命令白名单。
- 明确禁止命令清单。
- 明确执行人。
- 明确审批人。
- 明确执行窗口。
- 明确输出脱敏规则。
- 明确失败停止条件。
- 明确不得读取或输出完整节点链接。
- 明确不得触发任何写操作。
- 明确不得触发正式 cutover。

未补齐以上事项并获得新的人工 Go 前，不得执行真实只读预检。

## 未来可申请的只读预检范围

未来可申请的只读预检范围包括：

- 8443 当前监听服务确认。
- `gost` 8443 是否仍作为正式 / 回退链路。
- `socat` 是否未接管 8443。
- `socat` 18443 是否仍作为候选链路。
- 服务器防火墙只读状态。
- systemd 服务只读状态。
- 进程只读状态。
- 端口监听只读状态。

当前阶段全部不执行。

## 8443 / 端口 / 防火墙提醒

如果未来涉及 8443 接管或监听端口变更，必须确认：

- 云服务器安全组 TCP 8443 是否放行。
- 云防火墙 TCP 8443 是否放行。
- 服务器防火墙 TCP 8443 是否放行。
- 当前监听 8443 的服务。
- 切换后监听 8443 的服务。
- `gost` 8443 回滚路径。

凡涉及新增、变更监听端口或 8443 接管，必须同步检查云服务器安全组 / 云防火墙 / 服务器防火墙放行对应 TCP 端口。

本阶段不执行任何 8443、端口或防火墙检查命令。
