# Stage 3.3.24 C Read-Only Preflight Execution Authorization

本阶段只记录 C 方案真实只读预检执行授权。本阶段不是正式 cutover，不执行远程命令，不触发后端任务，不修改正式节点链接，不改变生产链路状态。

## 当前阶段结论

- 本阶段是只读预检执行授权，不是正式 cutover。
- 本阶段授权未来进入真实只读预检执行阶段。
- 本阶段本身不执行任何远程命令。
- 本阶段本身不 SSH。
- 本阶段不读取或修改 `node.share_link`。
- 本阶段不触发后端任务。
- 本阶段不新增数据库迁移。
- 本阶段不新增监听端口。
- 本阶段不让 `socat` 接管 8443。
- 本阶段不关闭、停用、降级或替换 `gost` 8443。

## SSH 登录授权边界

- 未来真实只读预检执行阶段允许 SSH 登录服务器。
- SSH 登录仅限只读预检。
- SSH 登录不得执行写操作。
- SSH 登录不得读取或输出完整节点链接。
- SSH 登录不得写入、上传、保存 SSH Key 或 Passphrase。
- SSH 登录不得触发正式 cutover。
- 本阶段只记录授权，不实际 SSH。

## 远程只读命令白名单

白名单命令必须使用占位符，不得写真实 IP、真实账号、真实节点链接或真实密钥。

| 命令占位符 | 用途 | 边界 |
| --- | --- | --- |
| `ssh <SSH_USER>@<REMOTE_HOST> "<READ_ONLY_COMMAND>"` | 表示未来经授权的 SSH 只读命令形态 | `<READ_ONLY_COMMAND>` 必须来自白名单 |
| `ss -lntp` | 查看监听端口 | 只读查看 |
| `lsof -iTCP -sTCP:LISTEN` | 查看监听端口 | 只读查看 |
| `ps -ef` | 查看进程 | 只读查看 |
| `systemctl status <SERVICE_NAME>` | 查看服务状态 | 禁止 stop / restart / start |
| `systemctl is-active <SERVICE_NAME>` | 查看服务 active 状态 | 只读查看 |
| `systemctl is-enabled <SERVICE_NAME>` | 查看服务 enable 状态 | 只读查看 |
| `ufw status` 或等价只读防火墙查看 | 查看服务器防火墙状态 | 禁止修改规则 |
| `iptables` / `nftables` 只读列举命令 | 查看规则状态 | 禁止新增、删除、修改 |
| `curl` / `nc` / `tcping` 类型只读连通性检查 | 查看连通性 | 仅限审批范围内目标 |
