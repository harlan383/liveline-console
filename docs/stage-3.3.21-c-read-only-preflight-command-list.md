# Stage 3.3.21 C Read-Only Preflight Command List

本阶段只从 Stage 3.3.20 command authorization matrix 中拆出未来可申请审批的只读预检命令清单。本阶段不是正式 cutover，不连接服务器，不执行远程命令，不触发后端任务，不修改业务代码，不修改正式节点链接。

## 当前阶段结论

- 本阶段是 C read-only preflight command list。
- 本阶段不是正式 cutover。
- 当前仍保持 No-Go。
- 本阶段只整理未来只读预检命令清单，不执行任何命令。
- 本阶段不修改 `node.share_link`。
- 本阶段不新增数据库迁移。
- 本阶段不新增监听端口。
- 本阶段不执行远程命令。
- 本阶段不触发后端任务。
- 本阶段不关闭、停用或降级 `gost` 8443。
- 本阶段不让 `socat` 接管 `8443`。

## 只读预检命令总原则

- 当前所有远程命令仍然禁止执行。
- 本阶段只列出未来可申请审批的只读命令。
- 未获得人工 Go 前不得执行。
- 未列入白名单的命令一律禁止。
- 命令输出必须脱敏。
- 不得输出完整节点链接、SSH Key、Passphrase、token、服务器密码、Reality privateKey。
- 只读预检命令清单只是未来审批依据，不代表本阶段允许执行。

| 命令 | 用途 | 当前是否允许 | 限制 |
| --- | --- | --- | --- |
| `git status` | 查看本地工作区状态 | 是 | 仅限本地仓库 |
| `git diff` | 查看本地文件差异 | 是 | 不输出完整节点链接或真实凭据 |
| `git log` | 查看提交历史 | 是 | 不接触服务器 |
| `git branch` | 查看当前分支 | 是 | 不触发任务 |
| `grep` / `rg` 文档检查 | 检查文档和代码文本 | 是 | 不读取 `.env`、私钥、真实日志 |
| 敏感信息扫描 | 检查密钥、token、完整链接风险 | 是 | 仅限仓库文件 |
| README 阶段状态检查 | 确认 README 阶段状态一致 | 是 | 仅限本地文档 |
| docs 阶段文档存在性检查 | 确认阶段文档是否存在 | 是 | 仅限本地文档目录 |

这些命令只允许用于本地仓库检查，不连接服务器，不触发后端任务，不读取真实数据库。

## 8443 只读预检清单

未来如涉及 8443，至少需要确认：

- 当前监听 8443 的服务。
- `gost` 8443 是否仍可作为回退链路。
- `socat` 是否未接管 8443。
- 云服务器安全组 TCP 8443 放行状态。
- 云防火墙 TCP 8443 放行状态。
- 服务器防火墙 TCP 8443 放行状态。

凡涉及新增、变更监听端口或 8443 接管，必须同步检查云服务器安全组 / 云防火墙 / 服务器防火墙放行对应 TCP 端口。
