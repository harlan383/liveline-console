# Stage 3.3.20 C Command Authorization Matrix

本阶段只整理 C 方案未来可能使用的命令授权矩阵。本阶段不是正式 cutover，不连接服务器，不执行远程命令，不触发后端任务，不修改业务代码，不修改正式节点链接。

## 当前阶段结论

- 本阶段是 C command authorization matrix。
- 本阶段不是正式 cutover。
- 当前仍保持 No-Go。
- 本阶段只整理未来命令授权矩阵，不执行任何命令。
- 本阶段不修改 `node.share_link`。
- 本阶段不新增数据库迁移。
- 本阶段不新增监听端口。
- 本阶段不执行远程命令。
- 本阶段不触发后端任务。
- 本阶段不关闭、停用或降级 `gost` 8443。
- 本阶段不让 `socat` 接管 `8443`。

## 命令授权总原则

- 默认所有远程命令均为禁止。
- 只有获得新的人工 Go 后，才允许进入命令授权审批。
- 命令必须分为：
  - 本地只读命令。
  - 远程只读命令。
  - 远程执行命令。
  - 数据库命令。
  - 后端任务命令。
  - 禁止命令。
- 每条命令必须明确目的、风险、审批人、执行窗口、回滚关联和输出脱敏要求。
- 未列入白名单的命令一律禁止。
- 命令授权矩阵只是未来审批依据，不代表本阶段允许执行。

## 本地只读命令矩阵

| 命令 | 用途 | 当前是否允许 | 限制 |
| --- | --- | --- | --- |
| `git status` | 查看工作区状态 | 是 | 仅限本地仓库 |
| `git diff` | 查看本地改动 | 是 | 不输出真实完整节点链接 |
| `git log` | 查看提交历史 | 是 | 不接触服务器 |
| `grep` / `rg` 文档检查 | 检查文档和代码文本 | 是 | 不读取 `.env`、私钥、真实日志 |

这些命令仅限本地仓库检查，不接触服务器，不触发后端任务，不读取真实数据库。

## 远程执行命令矩阵

| 命令类别 | 示例占位命令 | 当前状态 | 未来申请条件 | 回滚要求 |
| --- | --- | --- | --- | --- |

| `socat` 接管 8443 | `<SOCAT_TAKEOVER_8443>` | 当前禁止 | 正式 Go、8443 检查、回滚路径、客户端回退 | 必须能回退到 `gost` 8443 或原正式链路 |
| `gost` 停用 / 降级 / 替换 | `<GOST_8443_CHANGE>` | 当前禁止 | 正式 Go、回退链路确认、执行窗口 | 必须能恢复 `gost` 8443 |
| 防火墙修改 | `<FIREWALL_CHANGE>` | 当前禁止 | 正式 Go、云安全组和服务器防火墙审批 | 必须能撤销变更 |

未来只有正式 Go、执行窗口、回滚方案、负责人全部确认后，才可申请远程执行命令。

## 8443 端口相关命令矩阵

- 当前不允许执行任何 8443 检查或切换命令。
- 未来如涉及 8443 接管或监听端口变更，必须确认：
  - 云服务器安全组 TCP 8443 是否放行。
  - 云防火墙 TCP 8443 是否放行。
  - 服务器防火墙 TCP 8443 是否放行。
  - 当前监听 8443 的服务。
  - 切换后监听 8443 的服务。
  - `gost` 8443 回滚路径。
- 凡涉及新增、变更监听端口或 8443 接管，必须同步检查云服务器安全组 / 云防火墙 / 服务器防火墙放行对应 TCP 端口。
- 本阶段不新增监听端口。
- 本阶段不执行 8443 检查或切换命令。

## 后端任务命令矩阵

- 当前不允许触发任何后端任务。
- 未来如需触发，必须明确：
  - task_type。

- 未审批的数据库写入。
- 未审批的 `node.share_link` 修改。
- 未审批的后端任务触发。
- 删除日志。
- 清空历史。
- 覆盖备份。
- 写入真实密钥。
- 输出完整节点链接。
- 任何未列入白名单的命令。
