# Stage 3.3.9 Cutover Readiness Check

本阶段只做正式 cutover 准备度检查文档。本阶段不是正式 cutover，不做远程服务器变更，不修改业务代码，不修改正式节点链接。

## 当前状态

- 当前尚未正式 cutover。
- `node.share_link` 未修改。
- `gost` 8443 仍是当前正式/回退链路。
- `socat` 18443 仍是候选正式链路。
- Stage 3.3.6 前端运行验收通过。
- Stage 3.3.8 已提供客户端候选链接验收流程。
- Docker compose 验收仍受本地 Docker Buildx 权限限制，尚未完成完整 Docker 部署验收。

当前系统仍处于低风险方案 B：前端展示并复制候选链接，但不写正式数据、不触发任务、不连接服务器。

## 正式 cutover 阻塞项

当前至少存在以下阻塞项：

- Docker compose 验收仍受本地 Docker Buildx 权限限制，未完成完整 Docker 部署验收。
- 尚未记录真实客户端候选链接测试结果。
- 尚未形成正式 cutover 回滚执行清单。
- 尚未决定是否修改 `node.share_link`。
- 尚未决定是否让 `socat` 接管 `8443`。
- 尚未确认正式 cutover 的操作窗口。
- 尚未确认如果涉及 `8443` 服务切换时，云服务器安全组/云防火墙 TCP 8443 是否已放行。
- 尚未确认如果涉及 `8443` 服务切换时，服务器防火墙是否放行 TCP 8443。

以上阻塞项解除前，不应进入正式 cutover。

## 方案选择建议

当前不建议直接进入方案 C。

建议先完成：

1. 真实客户端候选链接测试记录。
2. `gost` 8443 回退链路可用性复核。
3. Docker compose 或等价完整运行环境验收。
4. 正式 cutover 回滚执行清单。
5. 是否写 `node.share_link` 的明确决策。
6. 是否让 `socat` 接管 `8443` 的明确决策。

如果未来涉及 `8443` 服务切换，必须再次确认云服务器安全组/云防火墙 TCP 8443 放行，并检查服务器防火墙是否放行。
