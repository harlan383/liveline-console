## 回滚原则

如未来 C 方案执行异常，应立即回滚到 `gost` 8443 或原正式链路。

回滚原则：

- 如果 `socat` 18443 或 `socat` 接管 8443 后异常，恢复 `gost` 8443 或原正式链路。
- 如果 `node.share_link` 修改后异常，恢复旧值。
- 如果远端服务异常，停止切换并恢复原状态。
- 如果云服务器安全组 / 云防火墙 / 服务器防火墙导致不可达，停止切换并恢复回退链路。
- 回滚后必须再次验证客户端可用性。
- 回滚记录不得包含完整节点链接、SSH Key、Passphrase、token、服务器密码或 Reality privateKey。
