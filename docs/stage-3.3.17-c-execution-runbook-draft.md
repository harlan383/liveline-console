# Stage 3.3.17 C Execution Runbook Draft

本阶段只新增 C 方案执行级 runbook 草案。本阶段不是正式 cutover，不连接服务器，不执行远程命令，不触发后端任务，不修改业务代码，不修改正式节点链接。

## 当前阶段结论

- 本阶段是 C 方案执行级 runbook 草案。
- 本阶段不是正式 cutover。
- 当前仍保持 No-Go。
- 本阶段只写草案，不执行任何步骤。
- 本阶段不修改 `node.share_link`。
- 本阶段不新增数据库迁移。
- 本阶段不新增监听端口。
- 本阶段不执行远程命令。
- 本阶段不触发后端任务。
- 本阶段不关闭、停用或降级 `gost` 8443。
- 本阶段不让 `socat` 接管 `8443`。

## C 方案执行级 runbook 草案范围

本草案只整理未来正式 cutover 的执行结构：

- 只描述阶段、检查点、停止条件、验证方式和回滚结构。
- 不写入真实节点链接。
- 不写入真实 SSH Key / Passphrase / token。
- 不写入真实服务器密码。
- 不写入真实服务器 IP。
- 不读取或修改真实数据库。
- 不执行远程命令。
- 不触发后端任务。

所有涉及真实对象的内容必须使用占位符，例如 `<REMOTE_HOST>`、`<SERVICE_NAME>`、`<OLD_SHARE_LINK>`、`<NEW_SHARE_LINK>`。

## 执行前硬性前置条件

正式执行前必须满足：

1. 获得新的人工 Go。
2. 明确是否允许修改 `node.share_link`。
3. 明确是否允许远程命令。
4. 明确是否允许 `socat` 接管 8443。
5. 明确是否允许关闭、停用、降级或替换 `gost` 8443。
6. 明确执行窗口。
7. 明确执行人、验收人、回滚负责人。
8. 明确回滚方案。
9. 明确客户端快速回退方式。
10. 明确验收记录模板。
11. 确认不得把完整节点链接或敏感凭据写入文档、日志或报告。

任何一项未满足时，必须保持 No-Go。

## 端口与防火墙前置检查

如果未来涉及 8443 接管或监听端口变更，必须确认：

- 云服务器安全组 TCP 8443 已放行。
- 云防火墙 TCP 8443 已放行。
- 服务器防火墙 TCP 8443 已放行。
- 当前监听 8443 的服务已确认。
- 切换后监听 8443 的服务已确认。
- `gost` 8443 回滚路径已确认。

凡涉及新增、变更监听端口，必须同步检查云服务器安全组 / 云防火墙 / 服务器防火墙放行对应 TCP 端口。

本阶段不执行任何端口或防火墙检查命令。

## 未来正式执行流程草案

### Phase 0：人工 Go 确认

目标：

- 确认 C 方案正式执行获得人工 Go。
- 确认执行窗口、负责人和回滚负责人。
- 确认本 runbook 已升级为执行级版本。

停止条件：

- 未获得人工 Go。
- 审批结论仍为 No-Go。
- 执行人、验收人或回滚负责人未确认。

### Phase 1：只读状态确认

目标：

- 只读确认当前 active node。
- 只读确认当前 `node.share_link` 是否存在。
- 只读确认当前 active routes。
- 只读确认 `gost` 8443 回退链路状态。
- 只读确认 `socat` 候选链路状态。

禁止：

- 不修改数据库。
- 不触发后端任务。
- 不执行远程命令，除非该阶段未来单独获得授权。

### Phase 2：旧链路和 node.share_link 备份确认

目标：

- 如果涉及 `node.share_link`，先备份 `<OLD_SHARE_LINK>`。
- 准备 `<NEW_SHARE_LINK>`。
- 确认不会在文档、日志或报告中写入完整链接。
- 准备恢复 `<OLD_SHARE_LINK>` 的回滚方案。

停止条件：

- `<OLD_SHARE_LINK>` 未备份。
- `<NEW_SHARE_LINK>` 未确认。
- 发现完整链接写入文档或日志的风险。

### Phase 3：执行窗口确认

目标：

- 确认开始时间。
- 确认结束时间。
- 确认观察时间。
- 确认避开直播时段。
- 确认客户端可快速回退。

停止条件：

- 执行窗口未确认。
- 当前处于直播或关键业务时段。
- 回滚负责人不在线。

### Phase 4：正式切换步骤草案

目标：

- 根据已批准方案执行正式切换。
- 如果涉及 `node.share_link`，必须先备份旧值，再写入新值，并准备恢复旧值方案。
- 如果涉及 8443 接管，必须先确认 `gost` 8443 回滚路径。

占位符：

```text
<REMOTE_HOST>
<SERVICE_NAME>
<OLD_SHARE_LINK>
<NEW_SHARE_LINK>
<ROLLBACK_SERVICE_NAME>
<APPROVED_COMMAND_LIST>
```

禁止：

- 当前阶段不得执行以上占位步骤。
- 不得写入真实 IP。
- 不得写入真实密钥。
- 不得写入完整节点链接。
- 不得写入服务器密码。
- 不得写入 token。

### Phase 5：客户端验收

目标：

- Shadowrocket 验证。
- v2rayN 验证。
- 软路由验证。
- 普通网页访问验证。
- 目标平台访问验证。
- 回退链路验证。

要求：

- 不记录完整节点链接。
- 不记录 SSH Key / Passphrase / token。
- 不记录服务器密码。

### Phase 6：观察期

目标：

- 观察短时间稳定性。
- 观察直播影响。
- 观察客户端断连情况。
- 观察回退链路是否仍可用。

停止条件：

- 直播出现明显影响。
- 客户端频繁断连。
- 目标平台访问异常。
- 回退链路不可用。

### Phase 7：完成记录或回滚

目标：

- 如果验收通过，记录最终结论。
- 如果任一关键验证失败，立即进入回滚。
- 回滚后重新验证客户端可用性。

记录要求：

- 不写完整节点链接。
- 不写真实 SSH Key。
- 不写 Passphrase。
- 不写 token。
- 不写服务器密码。

## 正式切换步骤草案

本节只写流程，不写真实命令。

1. 确认人工 Go。
2. 确认执行窗口。
3. 确认执行人、验收人、回滚负责人。
4. 只读确认当前状态。
5. 备份 `<OLD_SHARE_LINK>`。
6. 准备 `<NEW_SHARE_LINK>`。
7. 如果涉及 `node.share_link`，按批准范围写入 `<NEW_SHARE_LINK>`。
8. 如果涉及 8443 接管，按批准范围切换 `<SERVICE_NAME>`。
9. 执行客户端验收。
10. 进入观察期。
11. 成功则记录完成。
12. 失败则执行回滚。

如果涉及 `node.share_link`，必须先备份旧值，再写入新值，并准备恢复旧值方案。

如果涉及 8443 接管，必须先确认 `gost` 8443 回滚路径。

## 验证步骤草案

正式 cutover 验证应包含：

- Shadowrocket 验证。
- v2rayN 验证。
- 软路由验证。
- 普通网页访问验证。
- 目标平台访问验证。
- 直播影响观察。
- 回退链路验证。
- 敏感信息检查。

验证记录不得包含完整节点链接。

## 回滚步骤草案

回滚原则：

- `node.share_link` 异常时恢复 `<OLD_SHARE_LINK>`。
- `socat` 18443 或 `socat` 接管 8443 异常时回滚到 `gost` 8443 或原正式链路。
- 远端服务异常时停止切换并恢复原状态。
- 云安全组 / 云防火墙 / 服务器防火墙导致不可达时立即停止 cutover 并回滚。
- 回滚后必须重新验证客户端可用性。

回滚记录不得包含完整节点链接、SSH Key、Passphrase、token、服务器密码或 Reality privateKey。

## 失败判定条件

任一条件出现时应判定失败并停止推进：

- 客户端无法连接。
- Shadowrocket / v2rayN / 软路由任一关键场景失败。
- 普通网页访问失败。
- 目标平台访问异常。
- 直播出现明显影响。
- 8443 不可达。
- 回退链路不可用。
- 出现任何敏感信息写入风险。

## 停止条件

以下任一条件存在时必须停止：

- 未获得人工 Go。
- 执行窗口未确认。
- 执行人 / 验收人 / 回滚负责人未确认。
- `node.share_link` 旧值未备份。
- 远程命令未授权。
- 云服务器安全组 / 云防火墙 / 服务器防火墙未确认。
- `gost` 8443 回退链路不可用。
- 客户端无法快速回退。
- 发现真实密钥、完整链接、token、服务器密码写入风险。
